26 // [03/14/07] The file dialogs need refactoring, and will be refactored in Jugg.
27 // Things that might need to change:
28 // - The interface is not in fact platform agnotsic, it is win32 oriented.
29 // - Filter format is highly windows specific, and is a little fragile, both for
30 // win32 and for other platforms.
31 // - Platform specific path strings are exposed to the console, because the
32 // protected validators save them as such.
33 // - Several of the FDS_XXX values are not options we want to give the user, such
34 // as NOT warning on file overwrite. The values FDS_OVERWRITEPROMPT,
35 // FDS_MUSTEXIST, and FDS_CHANGEPATH are not good things to give the user.
36 // - The Execute method is virtual for good reason. It should be implemented for
37 // each subclass. If common behavior is needed for Execute(), it can be
38 // factored out in hidden platform specific code.
